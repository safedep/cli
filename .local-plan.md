# Cloud query: design plan

## Naming shape
- **Choice: flat.** `safedep query exec` and `safedep query schema get`.
- Rationale: IA lists `query`, `package`, `tenant`, `api-key`, `project`, `exclusion` as peer top-level domains; no `cloud` umbrella. gh/vercel precedent (no umbrella for cloud-side domains). Adding `cloud` forces every management-plane invocation one level deeper without adding meaningful shared identity. The data plane vs control plane split is implicit (both surfaces speak directly to `safedep <noun>`), and the existing convention test does not require an umbrella.

## Command tree
- `safedep query` (parent noun)
  - `safedep query exec` (verb=exec). Executes a SQL string.
  - `safedep query schema` (sub-noun)
    - `safedep query schema get` (verb=get). Fetches the SQL schema.

Verbs `exec` and `get` already in `internal/cmd/verbs.go`.

## SQL input (`query exec`)
- `--sql "<inline>"` (string flag, takes precedence)
- `--sql-file <path>` (reads from file when `--sql` empty)
- Stdin (when `--sql` and `--sql-file` empty AND stdin is not a TTY)
- Validation: trimmed, must be non-empty, max 64KB. Multi-statement detection: reject queries containing `;` followed by non-whitespace (vet does not, but cloud accepts a single statement; safer to reject).
- `--limit` int flag (default 100), maps to `PageSize`. Bound to [1, 10000].

## Result types (Renderable)
- `execResult { columns []string; rows []map[string]any; count int }`
  - JSON: `{ "columns": [...], "rows": [{col: val, ...}], "count": N }`
  - Plain: `col1\tcol2\n...` per row
  - Table: `dry/tui/table` with header row + body rows
- `schemaResult { schemas []schemaTable }` where `schemaTable { name string; columns []schemaColumn }` and column has name/description/selectable/filterable/required/reference_url
  - JSON: nested
  - Plain: per-column lines `<table>.<col>\t<flags>\t<ref>`
  - Table: per-column rows with badges

## Orchestration package
- `internal/cloudquery/` (new package, function-based)
  - `client.go`: `New(conn) *Service` taking `*grpc.ClientConn`
  - `service.go`: `Exec(ctx, in ExecInput) (*ExecResult, error)`, `Schema(ctx) (*Schema, error)`
  - `types.go`: `ExecInput`, `ExecResult`, `Schema`, etc.
  - Translates structpb to native Go types at the boundary.
- Cmd packages depend on a small interface they own (`type queryService interface{...}`) for testability with stubs.

## Exit codes
- 0 success
- 1 any failure (auth, bad args, validation, server-side error, missing creds). Server SQL errors print verbatim message to stderr via `tui.Error` and return non-nil so cobra exits 1.
